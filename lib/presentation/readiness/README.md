# Readiness UI State

## Purpose

This folder manages readiness checks shown to users before or during app use.

## What Is Here

- Controller logic for backend, model, or runtime readiness messaging.

## When To Edit This

- Edit this folder when readiness states or recovery paths change.
- Keep backend calls in infrastructure services.

## Related Folders

- `lib/infrastructure/app/` wires runtime services.
- `lib/presentation/screens/` displays readiness messages.
