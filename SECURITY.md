# Security Policy

## Supported Use

SACA is a research prototype. Security reports are still welcome, especially for:

- credential leaks,
- unsafe defaults,
- dependency vulnerabilities,
- local model/runtime path issues,
- accidental disclosure of private datasets or infrastructure details.

## Reporting a Vulnerability

Please do **not** open a public issue for sensitive security problems.

Instead:

1. prepare a short summary of the issue,
2. describe impact and reproduction steps,
3. include affected files, platforms, and versions,
4. contact the maintainers privately through the repository security/contact path.

## Response Expectations

We aim to acknowledge reports promptly and follow up with mitigation guidance,
fixes, or containment steps as appropriate for a research repository.

## Scope Notes

Because SACA is not a production clinical system, security fixes may be shipped
alongside documentation or workflow changes instead of full release branches.
